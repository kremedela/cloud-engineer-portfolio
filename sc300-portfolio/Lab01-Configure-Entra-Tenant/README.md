# Lab 01 – Configure and Manage a Microsoft Entra Tenant

## 🎯 Objective
Learn how to set up and manage a Microsoft Entra tenant, including tenant properties, user settings, group settings, and device configurations. This lab teaches foundational skills for identity and access management in Azure.

## 🧰 Azure Services Used
- Microsoft Entra ID
- Azure Portal
- PowerShell / Graph API
- Microsoft 365 Admin Center

## 📁 Folder Structure
- **docs/** – detailed walkthroughs, diagrams  
- **scripts/** – PowerShell/Graph scripts  
- **screenshots/** – images proving completion  

## 📝 Prerequisites
- Azure account  
- Global Administrator permissions  
- Modern web browser  

## 🧪 Lab Steps (Summary)
1. Access and review tenant properties  
2. Configure tenant properties (company name, contact, language, region)  
3. Configure user settings (password reset, app registration, external collaboration)  
4. Configure group settings (expiration, creation permissions, naming policies)  
5. Configure device settings (restrictions, device limit, MFA registration)  
6. Verify configuration using a test user and device  

## ✅ Expected Outcome
- Tenant properties configured with correct company info and regional settings  
- User settings optimized for security and collaboration  
- Group settings applied to manage group lifecycle and creation permissions  
- Device settings enforcing security policies and MFA requirements  
- Test user correctly assigned to group and registered a device  

## 🚀 Automation (Optional)
Use PowerShell or Graph API scripts to automate:  
- Viewing tenant details  
- Configuring user or group settings  
- Enforcing device policies  

Save any scripts in `/scripts/tenant-setup.ps1`.
